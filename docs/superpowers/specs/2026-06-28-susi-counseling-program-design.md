# 수시 입시 상담 통합 대시보드 — 설계 문서

작성일: 2026-06-28

## 1. 목적

진학 교사/상담자와 학생이 함께 사용하는 **수시 입시 상담 통합 대시보드**.
학생의 성적(내신·모의고사 등급)을 입력하면 대학·학과별 합격선과 자동 비교하여
합격 가능성을 진단하고, 수시 6장 지원 전략을 관리하며, 상담 기록을 누적한다.

## 2. 범위 (Scope)

포함:
- 학생 등록/선택 및 성적 입력
- 합격 가능성 자동 진단 (상향/적정/안정)
- 수시 6장 지원 보드 및 전략 균형 점검
- 상담 기록 관리
- 데이터 백업 (JSON 내보내기/불러오기)

제외 (YAGNI):
- 서버/계정/로그인 (단일 기기 localStorage 기반)
- 실제 합격선 데이터 수집 — 데이터는 사용자가 별도 제공 (본 프로젝트는 "틀"만 구축)
- 수능 정시 진단, 논술 채점 등

## 3. 기술 구조

- **단일 `index.html` 파일** — 현재 프로젝트 스타일 유지 (Noto Sans KR, 카드 UI)
- **Vanilla JS + localStorage** — 설치/서버 불필요, 브라우저에서 바로 작동
- **합격선 데이터**: 코드 내 JS 객체(`합격선DB`)로 내장. 사용자가 제공할 데이터를
  채워넣기만 하면 작동하는 구조
- 외부 의존성: 폰트(Google Fonts)만 사용, 그 외 라이브러리 없음

## 4. 화면 구성 (단일 페이지, 섹션/탭 전환)

1. **학생 선택 바 (상단 고정)** — 학생 추가/선택, 현재 학생 성적 요약 표시
2. **① 성적 입력** — 내신(전과목·주요과목 평균 등급), 모의고사 과목별 등급, 희망 계열/관심 학과
3. **② 합격 가능성 진단** — 학생 성적 vs `합격선DB` 자동 비교 → 지원 가능 대학·학과 리스트를
   상향/적정/안정 색상으로 분류 표시. 계열/전형으로 필터 가능
4. **③ 수시 6장 보드** — 6개 슬롯 카드(대학·학과·전형 배정), 상향·적정·안정 균형 게이지
5. **④ 상담 기록** — 날짜별 상담 내용·결정사항 누적 기록 (학생별)

## 5. 데이터 모델

### 5.1 합격선 DB (코드 내장, 사용자 제공 데이터의 틀)

```js
합격선DB = [
  {
    id: "snu-cse-gyogwa",   // 고유 식별자
    대학: "서울대학교",
    학과: "컴퓨터공학부",
    계열: "자연",            // 인문/자연/예체능
    전형유형: "학생부교과",   // 학생부교과/학생부종합/논술/실기
    모집인원: 30,
    합격선등급: 1.3,          // 등급 (1.0~9.0, 낮을수록 우수). 예: 50%컷
    수능최저: "3개합 6",      // 문자열, 없으면 ""
    비고: ""
  },
  // ...
]
```

### 5.2 학생 (localStorage)

```js
학생 = {
  id,
  이름,
  학년반,
  내신: { 전과목: 2.5, 주요과목: 2.3 },     // 등급
  모의: { 국: 2, 수: 3, 영: 1, 탐1: 2, 탐2: 3 }, // 등급
  희망: { 계열: "자연", 관심학과: ["컴퓨터공학부", ...] },
  지원: [ /* 6개 슬롯, 아래 참조 */ ],
  상담: [ { 날짜, 내용, 결정 } ]
}

// 지원 슬롯 (수시 6장)
지원슬롯 = {
  slot: 1,            // 1~6
  합격선id: "snu-cse-gyogwa",  // 합격선DB 참조 (직접 선택)
  분류: "상향",        // 진단 결과 자동 채움, 수정 가능
  메모: ""
}
```

전체 학생 목록은 `localStorage["susi.students"]`에 배열로 저장.
현재 선택 학생 id는 `localStorage["susi.currentStudentId"]`.

## 6. 진단 로직

학생의 **내신 등급**을 학과 **합격선 등급**과 비교 (등급은 낮을수록 유리).
비교 기준 등급은 기본적으로 `내신.주요과목` 사용 (코드 상단에서 전과목으로 변경 가능).

```
diff = 학생등급 - 합격선등급
  diff <= -0.3  → 안정 🟢  (합격선보다 0.3등급 이상 우수)
  -0.3 < diff <= +0.3 → 적정 🟡
  diff > +0.3   → 상향 🔴  (합격선보다 0.3등급 이상 부족)
```

- 임계값 `0.3`은 코드 상단 상수 `THRESHOLD`로 조정 가능
- 수능최저 충족 여부는 모의고사 등급 기준으로 참고 표시(경고)하되, 진단 등급에는 직접 반영하지 않음 (정보 제공 수준)

## 7. 수시 6장 전략 균형

- 6개 슬롯에 배정된 학과의 분류(상향/적정/안정) 집계
- 권장 균형 가이드 표시 (예: 상향 2 / 적정 2 / 안정 2). 실제 분포와 비교해 게이지로 시각화
- 균형이 한쪽으로 치우치면 안내 문구 표시

## 8. 데이터 백업

- 모든 학생 데이터는 localStorage에 자동 저장 (입력 즉시)
- **JSON 내보내기** 버튼: 전체 학생 데이터를 `.json` 파일로 다운로드
- **JSON 불러오기** 버튼: 파일 선택 → localStorage 복원 (기기 이전 대비)
- 합격선DB는 코드 내장이므로 백업 대상 아님

## 9. 코드 구성 (단일 파일 내 모듈 경계)

`index.html` 한 파일이지만 `<script>` 내부를 논리적 모듈로 구획:

- `data` — 합격선DB(내장), localStorage 입출력 (load/save students)
- `diagnose` — 진단 로직 (학생↔합격선 비교, 분류 반환)
- `render` — 섹션별 렌더링 (학생바, 성적입력, 진단, 6장보드, 상담)
- `events` — 입력/버튼 핸들러, 상태 변경 → 저장 → 재렌더

각 함수는 단일 책임을 갖고, 상태는 한 곳(`state`)에서 관리해 단방향 흐름 유지.

## 10. 테스트 전략

순수 로직(진단 분류, 균형 집계, 데이터 직렬화)을 부수효과 없는 함수로 분리하여
브라우저 콘솔 또는 간단한 인라인 테스트 블록으로 검증한다.
- `classify(학생등급, 합격선등급)` → "상향"/"적정"/"안정" 경계값 테스트
- export/import 라운드트립 (직렬화 후 역직렬화 시 동일 데이터)
