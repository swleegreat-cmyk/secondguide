# 작업 계획 규칙

작업을 받으면 바로 코딩하지 말고, 먼저 계획을 세워라.

## 계획 작성법

1. 작업을 3단계 이하로 분해한다.
2. 각 단계마다 완료 조건을 명시한다.
3. 한 단계가 끝나야 다음 단계로 넘어간다.

```
예시:
1. [버튼 컴포넌트 추가] → 완료: 버튼이 화면에 렌더링됨
2. [클릭 이벤트 연결] → 완료: 클릭 시 콘솔에 로그 출력
3. [스타일 적용] → 완료: 디자인 시안과 일치
```

## 큰 작업 분할

한 번에 50줄 이상 변경이 예상되면 반드시 분할한다.
- 파일 1개당 1작업으로 분리
- 각 작업은 독립적으로 검증 가능해야 함
- 이전 작업 결과를 확인한 후 다음 작업 진행
