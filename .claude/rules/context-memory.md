# 맥락 유지 규칙

AI는 이전 대화를 잊는다. 이를 방지하기 위한 규칙.

## 작업 시작 시

- 현재 프로젝트 상태를 먼저 확인한다 (파일 구조, 기존 코드 패턴).
- 사용자의 이전 요청과 현재 요청의 연결 관계를 파악한다.
- 기존 코드 스타일/패턴을 먼저 읽고 따른다.

## 작업 중

- 자신이 변경한 내용을 기억하고, 충돌하는 수정을 하지 않는다.
- 여러 파일을 수정할 때, 파일 간 의존성을 추적한다.
- 중간 결과를 사용자에게 보고한다.

## 작업 완료 시

- 무엇을 변경했는지 요약한다.
- 후속 작업이 필요하면 명시한다.
- 변경사항이 기존 기능에 영향을 줄 수 있는지 확인한다.
